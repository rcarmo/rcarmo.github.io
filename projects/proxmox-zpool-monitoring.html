Why it exists

Motivation

Proxmox has good VM-level metrics but its built-in ZFS monitoring is limited. I wanted per-pool capacity, I/O, and health trends in Grafana alongside the rest of the homelab metrics. This script fills that gap.

How it works

Architecture

A Python script that runs as a cron job on each Proxmox node. It calls zpool status, zpool iostat, and zfs list, parses the output, and sends the metrics to Graphite via the plaintext protocol. Grafana reads from Graphite.

Notable features

Highlights

No agent daemon — just a cron job and a Graphite endpoint
Exports pool health, capacity, read/write IOPS, and error counts
Tested on Proxmox 7.x and 8.x with single and multi-disk pools
