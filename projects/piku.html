About

piku is a single Python script installed as a git post-receive hook. Push your code, piku reads the Procfile, installs dependencies into a per-app virtualenv/gopath/node_modules, starts your workers under uwsgi, and wires them up behind nginx with optional Let's Encrypt TLS. The whole runtime fits in ~1500 lines of readable Python and runs on a first-generation Raspberry Pi.

Features
What it does
⌥
Heroku-style workflow
git remote add piku piku@yourserver:appname. git push piku main. That's the entire deploy flow.
🌐
Python, Node, Go, Clojure, Ruby, Java, PHP, static
If it runs in a shell it runs in piku. Language runtimes are detected from lock files and Procfiles.
🔒
Automatic TLS
Let's Encrypt certificates via ACME, virtual host routing, zero-downtime deploys, and static file serving — all configured through a single ENV file.
⬆
Live scaling and config
ps:scale web=3 and config:set KEY=val take effect instantly without a redeploy or restart.
🐧
Any POSIX host
Runs on Debian, Ubuntu, Alpine, FreeBSD, WSL. Tested on ARM and Intel from a $35 Raspberry Pi to cloud VMs. Stable since 2016, maintained by the piku org.
